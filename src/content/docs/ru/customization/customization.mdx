В современном мире удобство и индивидуальность рабочего пространства важны как никогда. 
В дистрибутиве meowrch для этого создана мощная и гибкая система кастомизации, позволяющая управлять внешним видом и поведением множества приложений единой командой.

Центральным элементом этой системы является [Pawlette](/ru/customization/pawlette) — универсальный менеджер тем. 
Он объединяет все настройки в одном месте и позволяет плавно переключаться между темами, сохраняя при этом уникальные пользовательские правки и предпочтения. 
Применение тем охватывает оконные менеджеры, панели, терминалы, оболочки и другие ключевые компоненты вашей системы.

## Управление обоями

Для полного погружения в персонализацию рабочего стола meowrch предоставляет удобный скрипт для смены обоев:

```
~/.local/bin/set-wallpaper.sh
```

Этот скрипт автоматически выбирает правильный инструмент в зависимости от используемого сервера отображения — `feh` для **X11** или `swww` для **Wayland** — обеспечивая быстрый и беспроблемный способ обновления фона рабочего стола.

### Как установить текущие обои

В системе хранится символическая ссылка на активные обои, 
что позволяет легко восстановить или применить их повторно с помощью команды:

```
~/.local/bin/set-wallpaper.sh --current
```

Ссылка на текущие обои находится по адресу:

```
~/.local/share/wallpapers/.current.wall
```

### Добавление обоев, не привязанных к теме

Если вы хотите расширить коллекцию обоев и использовать их вне зависимости от выбранных тем, 
просто скопируйте изображения в директорию:

```
~/.local/share/wallpapers/
```

Для удобного выбора обоев можно использовать графический интерфейс на базе Rofi, доступный через скрипт:

```
~/.local/bin/rofi-menus/wallpaper-selector.sh
```

## Использование стандарта XDG

Meowrch активно поддерживает и продвигает стандарт XDG, который определяет, где должны храниться конфигурационные файлы, кэш и данные в системе Linux. Это обеспечивает:

- Чистоту домашней директории пользователя
- Унифицированный способ хранения файлов конфигурации для разных приложений
- Совместимость с большинством современных Linux-систем и утилит

### Что такое XDG?

XDG — это спецификация, которая определяет стандартные пути для хранения конфигураций, кэша, данных и других файлов приложений. Она помогает приложениям и пользователям организовывать файлы удобно и последовательно.

### XDG + Meowrch

В meowrch установлены следующие переменные окружения, влияющие на расположение файлов:

```bash
shenv ZDOTDIR "$XDG_CONFIG_HOME/zsh"
```

- Эта переменная указывает оболочке zsh, что ее конфигурационные файлы находятся в пути, соответствующем переменной окружения `$XDG_CONFIG_HOME/zsh`, что обычно означает `~/.config/zsh`.

```bash
shenv GTK2_RC_FILES "$XDG_CONFIG_HOME/gtk-2.0/gtkrc"
```

- Задает местоположение конфигурационного файла gtk2, чтобы приложения на GTK могли читать настройки внешнего вида из правильного каталога.

```bash
shenv XCURSOR_PATH "/usr/share/icons:$XDG_DATA_HOME/icons"
```

- Устанавливает пути поиска для тем курсоров мыши, объединяя системный каталог и пользовательскую папку с иконками.

```bash
shenv CARGO_HOME "$XDG_DATA_HOME/cargo"
```

- Путь для данных менеджера пакетов Rust (cargo), соответствующий стандарту XDG.

```bash
shenv CUDA_CACHE_PATH "$XDG_CACHE_HOME/nv"
```

- Путь для кэша Nvidia CUDA.

```bash
shenv GNUPGHOME "$XDG_DATA_HOME/gnupg"
```

- Директория для ключевых файлов GnuPG.

```bash
shenv REDISCLI_HISTFILE "$XDG_DATA_HOME/redis/rediscli_history"
```

- Путь для истории команд клиента Redis.

```bash
shenv RUSTUP_HOME "$XDG_DATA_HOME/rustup"
```

- Директория для утилиты Rustup.

```bash
shenv NODE_REPL_HISTORY "$XDG_DATA_HOME/node_repl_history"
```

- Файл истории для NodeJS REPL.

```bash
shenv PYENV_ROOT "$XDG_DATA_HOME/pyenv"
```

- Путь для менеджера версий Python pyenv.

```bash
shenv WAKATIME_HOME "$XDG_CONFIG_HOME/wakatime"
```

- Директория конфигурации для плагина Wakatime.

---

### Почему это важно?

- Благодаря работе через стандарт XDG ваши настройки и конфигурации всегда хранятся в однородных и предсказуемых местах.
- Это упрощает резервное копирование, синхронизацию и совместное использование настроек между различными машинами.
- Улучшает совместимость с другими Linux-дистрибутивами и программами, использующими тот же стандарт.
